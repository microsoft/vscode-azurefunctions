# PR and Release Creation

## Step 1: Stage and Commit

```bash
git add package.json CHANGELOG.md
git commit -m "Release prep for <NEW_VERSION>"
```

---

## Step 2: Push Branch

```bash
git push origin <BRANCH_NAME>
```

---

## Step 3: Create Draft PR

```bash
gh pr create \
  --draft \
  --title "Release prep for <NEW_VERSION>" \
  --body "## Release prep for <NEW_VERSION>

> 🤖 This PR was generated by GitHub Copilot. Please review all changes before merging."
```

Print the PR URL to the chat.

---

## Step 4: Create Draft Release

Use the approved changelog entry as the release body, but strip the `## X.Y.Z - YYYY-MM-DD` heading line — the title carries the version and GitHub stamps the date on publish.

```bash
gh release create "v<NEW_VERSION>" \
  --draft \
  --title "v<NEW_VERSION>" \
  --notes "<CHANGELOG_ENTRY_BODY>"
```

`CHANGELOG_ENTRY_BODY` = the changelog entry with the date stripped from the heading — keep `## X.Y.Z`, remove ` - YYYY-MM-DD`. GitHub stamps the date on publish.

---

## Step 5: Create Alpha Version Bump PR

This PR is stacked on the release branch so that once the release PR merges into `main`, this becomes an immediate candidate to bump `main` to the next development version.

### 5.1 Create alpha bump branch

The release branch is still checked out from Step 2, so branch directly from it:

```bash
git checkout -b copilot/alpha-bump/v<NEW_VERSION>
```

### 5.2 Bump the version

```bash
npm version prerelease --preid=alpha
```

This updates both `package.json` and `package-lock.json` and creates a git commit automatically.

### 5.3 Push

```bash
git push origin HEAD
```

### 5.4 Create draft PR targeting the release branch

```bash
gh pr create \
  --draft \
  --base <BRANCH_NAME> \
  --title "Bump version after release" \
  --body "## Bump version after release

Bumps version in preparation for the next development cycle.

> ⚠️ **Do not merge until the upstream release PR has been merged and the extension has been published.**
> This PR depends on <RELEASE_PREP_PR_URL> — merge that first.

> 🤖 This PR was generated by GitHub Copilot. Please review all changes before merging."
```

Print the PR URL to the chat.

Print the release URL to the chat.
